// STRYKE-NEO4J — ENGINEERING REPORT

Documentation GitHub

Driver + blocking facade

The client is neo4rs, the pure-Rust Bolt protocol implementation — no JVM, no official-driver FFI. neo4rs is async, so the cdylib owns one multi-thread tokio runtime and block_ons each call. From stryke's perspective every Neo4j::* function is an ordinary blocking call; the async machinery is entirely internal.

Graph pool cache

neo4rs Graph is itself a connection pool. One is cached per (uri, user, database) tuple, so repeated queries reuse pooled Bolt connections. A call that errors evicts its pool from the cache, so the next call reconnects cleanly rather than reusing a poisoned session.

Rows to JSON

neo4rs deserializes a Bolt row directly into serde_json::Value, so the conversion is one call per row: scalars become JSON scalars, a returned node or relationship becomes its property map, and lists/maps nest naturally. Arbitrary RETURN shapes round-trip without the caller declaring a schema, and the result is keyed by the return aliases.

Parametrized Cypher

Statements use $name placeholders; JSON params (bool / integer / float / string) are bound positionally by name, so values never enter the query text — no string interpolation, no injection surface. run executes a write/DDL statement that returns no rows; query / scalar stream and collect results.

Dependencies

neo4rs (Bolt driver), tokio (runtime), serde_json for the FFI payloads and row deserialization, once_cell for the runtime + pool cache. Pure-Rust throughout — no JVM, no native client library.
